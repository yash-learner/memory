## Learnings

- **`??` vs `||` — empty string passes through `??`:** when an API sends `""` instead of `null` for "not set", `??` won't fall back. Normalize with `|| undefined` first. Full write-up: [`learnings/typescript/nullish-coalescing-vs-or.md`](../../../learnings/typescript/nullish-coalescing-vs-or.md).
- **`const element = (...)` vs `function Component()`:** when to inline JSX as a value vs extract a component. Start with `const`, promote when you need props/hooks/reuse. Full write-up: [`learnings/react/jsx-const-vs-component.md`](../../../learnings/react/jsx-const-vs-component.md).


- **CSS cascade layers + `!important` interact in counterintuitive ways.** Full write-up: [`learnings/css/cascade-layers.md`](../../../learnings/css/cascade-layers.md).
- The cleanest fix was often not "fight the `!important` rule" but **avoid matching its selector** — by changing the configured logo's `alt` so it no longer matched `[alt*="logo" i]`. Same pattern the existing `headerImage` (`alt="Custom Header"`) had been quietly using all along.
- DevTools' "Emulate CSS media type: print" (Rendering tab) is the right tool when something behaves differently in print vs screen — saves a million test prints.
